# Abyss Breaker — Project Specification

## 1. 프로젝트 개요

**Abyss Breaker**는 모바일 세로 화면에 최적화된 실시간 벽돌깨기 로그라이크 웹게임이다.

게임의 기본 형태는 고전적인 브레이크아웃·아르카노이드 방식이다.

플레이어는 화면 하단의 막대를 좌우로 움직여 공을 반사하고, 상단의 벽돌을 모두 파괴한다. 벽돌을 파괴하면 일정 확률로 아이템이 떨어지며, 막대로 아이템을 획득해 현재 스테이지를 유리하게 진행한다.

향후에는 스테이지 사이 능력 선택, 특수 벽돌, 보스, 유물, 영구 성장 등을 추가한다.

개발 목표는 기능을 많이 넣는 것이 아니라, 막대 조작과 공 반사가 안정적이고 반복해서 플레이할 수 있는 코어 게임을 만드는 것이다.

---

## 2. 핵심 플레이 흐름

```text
게임 시작
→ 공이 막대 위에서 대기
→ 화면 터치 또는 발사 조작
→ 공이 벽, 막대, 벽돌에 반사
→ 벽돌 HP 감소
→ 벽돌 파괴
→ 점수 획득
→ 일정 확률로 아이템 낙하
→ 막대로 아이템 획득
→ 멀티볼, 막대 확장, 공 감속 효과 적용
→ 모든 파괴 가능한 벽돌 제거
→ 스테이지 클리어
→ 다음 스테이지 또는 능력 선택
```

공이 화면 아래로 떨어지면 해당 공은 제거된다.

멀티볼 상태에서는 다른 공이 남아 있는 동안 게임이 계속된다.

마지막 공까지 모두 떨어지면 생명 1개를 잃는다.

생명이 남아 있으면 공을 막대 위에 다시 배치하고 플레이를 재개한다.

생명이 0이 되면 게임오버다.

## 3. 개발 우선순위

기능과 구조를 결정할 때 다음 순서를 따른다.

1. 막대 조작이 정확하고 반응성이 좋은가
2. 공과 막대의 반사가 자연스러운가
3. 공이 벽돌을 통과하거나 내부에 갇히지 않는가
4. 모바일 세로 화면에서 플레이하기 편한가
5. 아이템 획득이 플레이에 변화를 주는가
6. 스테이지를 반복해서 플레이할 이유가 있는가
7. 모바일 성능이 안정적인가
8. 시각 효과와 추가 콘텐츠가 코어 플레이를 방해하지 않는가

코어 조작과 물리 안정성을 해치는 기능은 추가하지 않는다.

## 4. 기존 방식에서 제거한 구조

이 프로젝트는 더 이상 턴제 공 발사 게임이 아니다.

다음 구조는 사용하지 않는다.

* 드래그 조준
* 조준선
* 손을 놓아 공 발사
* 여러 공의 순차 발사
* 공이 하단으로 복귀하는 구조
* 모든 공 복귀 후 턴 종료
* 턴마다 벽돌이 한 칸 하강
* 위험선 도달
* 벽돌 공격력에 의한 플레이어 HP 감소
* 턴 정산
* 발사 대기열
* `aiming`, `launching`, `resolving` 상태
* BBTAN 방식의 웨이브 진행
* `inputLocked`를 이용한 턴 전환

기존 턴제 로직과 상태는 새 코드에 남기지 않는다.

## 6. 작업 환경

현재 프로젝트는 VS Code와 Codex를 이용해 수정한다.

GitHub 웹에서 파일 전체를 복사·붙여넣기 하는 방식으로 반영할 수 있으므로 다음 원칙을 지킨다.

* 기존 파일을 수정하기 전에 전체 내용을 확인한다.
* 요청한 범위 밖의 파일을 불필요하게 변경하지 않는다.
* 파일 전체 교체가 가능한 형태로 작성한다.
* 일부 코드를 `...`으로 생략하지 않는다.
* 존재하지 않는 함수, DOM ID, 전역 객체를 참조하지 않는다.
* 기존 정상 기능을 제거할 경우 이유를 명확히 기록한다.
* 사용하지 않는 옛 상태와 함수는 남기지 않는다.
* 변경 후 수정한 파일 목록과 테스트 방법을 보고한다.
* 브라우저 콘솔 오류가 없는지 점검한다.

## 9. 스크립트 로딩 순서

`index.html`의 `body` 종료 직전에 다음 순서로 로드한다.

```html
<script src="data.js"></script>
<script src="state.js"></script>
<script src="game.js"></script>
<script src="render.js"></script>
<script src="ui.js"></script>
<script src="main.js"></script>
```

이 순서를 임의로 변경하지 않는다.

모든 파일은 같은 전역 네임스페이스를 사용한다.

```javascript
window.AbyssBreaker = window.AbyssBreaker || {};
```

권장 구조:

```text
AbyssBreaker.Data
AbyssBreaker.State
AbyssBreaker.Game
AbyssBreaker.Render
AbyssBreaker.UI
AbyssBreaker.Main
```

전역 변수와 전역 함수를 무분별하게 만들지 않는다.

## 11. UI 원칙

* 전체 페이지 스크롤 금지
* 중첩 스크롤 금지
* Canvas가 화면 대부분을 차지
* HUD 높이를 최소화
* 하단 컨트롤 높이를 최소화
* 설명문과 카드 UI를 과도하게 배치하지 않음
* 플레이 중 필요한 정보만 표시
* 모바일 세로 화면 우선
* PC에서는 최대 너비 약 480px의 세로형 게임 화면으로 중앙 정렬
* 약 320px 너비의 작은 모바일 화면도 대응
* 노치와 홈 인디케이터 고려
* `100dvh` 우선 사용
* `100vh` fallback 제공
* `safe-area-inset-top`, `safe-area-inset-bottom` 적용
* Canvas에서 브라우저 스크롤과 확대 제스처 방지
* 버튼 터치 영역은 최소 약 44px 확보
* 숨겨진 오버레이는 클릭과 터치를 차단하지 않아야 함
* 오버레이가 표시된 동안 게임 입력은 차단

사용자에게 표시되는 문구는 한국어로 작성한다.

변수명, 함수명, 객체 키, DOM ID는 영어를 사용한다.

## 13. 게임 모드

게임 모드는 하나의 명확한 상태 값으로 관리한다.

권장 모드:

```text
boot
ready
playing
lifeLost
stageClear
paused
gameover
```

### 상태 설명

* `boot`: 초기 파일과 Canvas 준비 단계
* `ready`: 공이 막대 위에 있고 발사 대기 중
* `playing`: 공이 이동하고 벽돌을 파괴하는 상태
* `lifeLost`: 마지막 공을 놓쳐 생명을 잃은 상태
* `stageClear`: 모든 벽돌을 파괴한 상태
* `paused`: 일시정지 상태
* `gameover`: 생명이 모두 소진된 상태

한 번에 하나의 주요 모드만 활성화한다.

일시정지 전 모드는 별도로 저장해 정상적으로 복구한다.

## 15. 막대

막대는 화면 하단에 배치한다.

권장 데이터 구조:

```javascript
{
  x: 0,
  y: 0,
  width: 90,
  baseWidth: 90,
  height: 14,
  targetX: 0,
  speed: 0,
  expandTimeRemaining: 0
}
```

### 막대 이동

* 모바일에서는 Canvas 좌우 드래그로 이동
* PC에서는 마우스 이동 또는 드래그로 이동
* 포인터의 X 좌표를 기준으로 막대 중심을 이동
* 막대가 Canvas 좌우 경계를 벗어나지 않음
* 위치 값에 NaN이 들어가지 않게 방어
* 화면 크기 변경 시 위치를 유효 범위로 보정
* 필요하면 현재 위치와 목표 위치 사이를 보간해 부드럽게 이동
* 조작 지연이 크게 느껴지는 과도한 보간은 사용하지 않음

### 막대 확장

* 막대 확장 아이템 획득 시 너비 증가
* 최대 너비 제한
* 효과 종료 후 기본 너비로 복구
* 효과 중 다시 획득하면 지속 시간 갱신 또는 연장
* 너비 변경 후 화면 밖으로 나가지 않게 위치 보정

## 16. 공

각 공은 독립 객체로 관리한다.

권장 데이터 구조:

```javascript
{
  id: 0,
  x: 0,
  y: 0,
  prevX: 0,
  prevY: 0,
  vx: 0,
  vy: 0,
  radius: 6,
  active: true,
  attached: false,
  collisionCooldowns: {}
}
```

### 공 기본 규칙

* 공은 실시간으로 이동
* 좌측 벽, 우측 벽, 상단 벽에서 반사
* 화면 아래로 완전히 벗어나면 제거
* 하단에서는 반사하지 않음
* 공이 하나 이상 남아 있으면 플레이 계속
* 마지막 공이 사라지면 생명 감소
* 새 생명 시작 시 공 하나를 막대 위에 배치
* 대기 중인 공은 막대 이동을 따라감
* 발사 후에는 막대와 독립적으로 이동

### 공 속도

* 기본 속도는 `data.js`에서 정의
* 공마다 속도 크기를 일정 범위로 유지
* 벽이나 막대 충돌 후 속도가 비정상적으로 증가하지 않게 정규화
* 지나치게 수평으로만 움직이지 않도록 최소 수직 속도 보장
* 공의 Y 속도 절댓값이 최소값보다 작으면 보정
* 속도가 0, NaN, Infinity가 되면 안전한 기본 방향으로 복구

---

## 17. 공 발사

시작 전 공은 막대 위에 붙어 있다.

발사 방식:

* 모바일: 화면 탭 또는 발사 버튼
* PC: Canvas 클릭 또는 발사 버튼
* 막대 이동을 위한 드래그가 발사로 잘못 처리되지 않게 구분
* 일정 거리 이상 드래그한 경우 발사하지 않음
* `ready` 상태에서만 발사 가능
* 일시정지, 게임오버, 스테이지 클리어 중에는 발사 불가
* 발사 방향은 기본적으로 위쪽
* 완전 수직이 반복되지 않도록 약간의 좌우 각도를 줄 수 있음
* 발사 직후 공이 막대와 다시 충돌하지 않게 시작 위치 보정

---

## 18. 막대와 공 충돌

막대는 축 정렬 사각형, 공은 원으로 처리한다.

공이 아래쪽으로 이동 중일 때만 막대 충돌을 처리한다.

충돌 후 공을 막대 위로 위치 보정한다.

### 반사 각도

막대 중앙과 충돌 지점의 상대 위치를 사용한다.

```text
막대 중앙 충돌
→ 거의 수직으로 위쪽 반사

막대 왼쪽 충돌
→ 왼쪽 위 방향 반사

막대 오른쪽 충돌
→ 오른쪽 위 방향 반사

막대 끝부분 충돌
→ 더 큰 좌우 각도
```

권장 계산 개념:

```text
relativeHit = (ball.x - paddle.centerX) / (paddle.width / 2)
```

`relativeHit`은 -1에서 1 사이로 제한한다.

최대 반사 각도를 설정하고 이를 이용해 새 속도 방향을 계산한다.

반사 후에도 공의 전체 속도 크기는 유지한다.

### 막대 이동 영향

향후 막대 이동 속도를 공의 X 속도에 일부 반영할 수 있다.

v0.1에서는 구조만 확장 가능하게 두고, 과도한 가속은 적용하지 않는다.

---

## 19. 벽 충돌

* 좌측 벽 충돌 시 X 속도 양수로 보정
* 우측 벽 충돌 시 X 속도 음수로 보정
* 상단 벽 충돌 시 Y 속도 양수로 보정
* 충돌 후 공 위치를 벽 안쪽으로 보정
* 같은 프레임에 반복 반사되지 않게 처리
* 하단은 충돌이 아니라 공 손실 판정

## 20. 벽돌

벽돌은 상단 고정 그리드에 배치한다.

권장 데이터 구조:

```javascript
{
  id: 0,
  type: "normal",
  row: 0,
  col: 0,
  x: 0,
  y: 0,
  width: 0,
  height: 0,
  hp: 1,
  maxHp: 1,
  score: 100,
  dropChance: 0.1,
  destructible: true,
  alive: true,
  rewardGranted: false
}
```

### 벽돌 규칙

* 기본 벽돌 HP는 1
* 강화 벽돌은 HP 2 이상
* 현재 HP 또는 종류에 따라 색상 구분
* 공 충돌 시 HP 감소
* HP가 0 이하가 되면 파괴
* 파괴 시 점수 지급
* 일정 확률로 아이템 생성
* 파괴 보상은 한 번만 지급
* 모든 파괴 가능한 벽돌이 제거되면 스테이지 클리어
* 파괴 불가능한 벽돌은 클리어 조건에서 제외 가능

---

## 21. 벽돌 패턴

벽돌 패턴은 `data.js`에서 관리한다.

권장 방식:

```javascript
[
  "1111111",
  "1222221",
  "1111111"
]
```

예시 의미:

```text
0: 빈 공간
1: 일반 벽돌
2: 강화 벽돌
3 이상: 향후 특수 벽돌
```

그리드 관련 값:

* 열 수
* 행 수
* 좌우 여백
* 벽돌 간격
* 상단 시작 위치
* 벽돌 너비
* 벽돌 높이

벽돌 크기와 위치는 하나의 공통 계산 로직을 사용한다.

여러 파일에서 서로 다르게 계산하지 않는다.

---

## 22. 공과 벽돌 충돌

공은 원, 벽돌은 축 정렬 사각형으로 처리한다.

권장 방식:

* 원과 사각형의 최근접점 계산
* 충돌 거리와 공 반지름 비교
* 이전 위치와 침투 깊이를 이용해 충돌 면 판단
* X 또는 Y 방향 속도 반전
* 모서리 충돌 시 충돌 법선 기반 반사 가능
* 충돌 후 공을 벽돌 밖으로 위치 보정

### 중복 충돌 방지

* 동일 공이 같은 벽돌에 한 프레임 동안 여러 번 피해를 주지 않음
* 짧은 충돌 쿨다운이나 프레임 단위 충돌 집합 사용
* 파괴된 벽돌은 즉시 충돌 대상에서 제외
* 보상과 아이템 드롭은 한 번만 처리

### 빠른 공 대응

공 속도가 빨라져도 벽돌을 통과하지 않도록 서브 스텝을 사용한다.

권장 방식:

```text
한 프레임의 이동 거리 계산
→ 허용 이동 거리보다 크면 여러 단계로 분할
→ 각 단계마다 벽과 벽돌 충돌 확인
```

서브 스텝 최대 횟수를 제한해 모바일 성능을 보호한다.

## 23. 생명 시스템

기본 생명은 3개다.

### 공 손실

* 공이 화면 아래로 완전히 벗어나면 해당 공 제거
* 다른 공이 남아 있으면 게임 계속
* 마지막 공이 제거됐을 때만 생명 1 감소
* 생명 감소 처리는 한 번만 실행

### 생명이 남은 경우

```text
마지막 공 손실
→ 생명 1 감소
→ 아이템과 임시 공 정리
→ 짧은 안내
→ 막대 초기 위치 복구
→ 공 1개를 막대 위에 배치
→ ready 상태
```

막대 확장이나 감속 효과를 생명 손실 후 유지할지는 데이터 설정으로 결정한다.

v0.1 기본 권장값:

* 멀티볼은 초기화
* 화면에 떨어지는 아이템 제거
* 막대 확장 효과 초기화
* 공 감속 효과 초기화

### 생명이 0인 경우

* 게임오버
* 게임 입력 차단
* 최고 점수 갱신
* 재시작 버튼 표시

## 25. 초기 아이템

### 25.1 막대 확장

* 일정 시간 막대 너비 증가
* 기본 권장 배율 1.5배
* 최대 너비 제한
* 화면 경계를 벗어나지 않음
* 다시 획득하면 지속 시간을 갱신하거나 연장
* 효과 종료 후 기본 너비 복구

### 25.2 멀티볼

* 현재 활성 공 하나를 기준으로 추가 공 2개 생성
* 원래 공과 다른 각도로 퍼져 나감
* 생성 공이 벽이나 막대 안에서 시작하지 않게 위치 보정
* 모든 공은 독립적으로 이동
* 한 공이 떨어져도 다른 공이 남으면 생명 감소 없음
* 최대 공 개수 제한
* 활성 공이 없는 상태에서는 적용하지 않거나 대기 공을 기준으로 처리

v0.1 권장 최대 공 개수:

```text
8개
```

### 25.3 공 감속

* 일정 시간 모든 공의 이동 속도를 감소
* 기본 권장 배율 0.75
* 너무 느려지지 않도록 최소 속도 유지
* 다시 획득하면 지속 시간 갱신 또는 연장
* 효과 종료 시 정상 속도로 복구
* 여러 번 속도를 곱해 원래 속도를 잃지 않도록 기본 속도와 효과 배율을 분리

## 26. 아이템 드롭

벽돌 파괴 시 다음 순서로 처리한다.

```text
벽돌 파괴
→ 점수 지급
→ 드롭 확률 판정
→ 아이템 종류 가중치 선택
→ 벽돌 중앙에서 아이템 생성
```

원칙:

* 한 벽돌은 아이템을 최대 한 개만 드롭
* 드롭 확률은 `data.js`에서 관리
* 아이템 종류 선택은 가중치 기반
* 아이템 최대 동시 개수 제한
* 화면에 너무 많은 아이템이 쌓이지 않게 제한
* 파괴 불가능 벽돌은 아이템을 드롭하지 않음

## 27. 아이템과 막대 충돌

아이템은 사각형으로 처리한다.

막대와 AABB 충돌하면 획득한다.

획득 순서:

```text
충돌 확인
→ 아이템 비활성화
→ 효과 적용
→ 획득 파티클 생성
→ 안내 텍스트 표시
→ 배열에서 제거
```

같은 아이템이 한 프레임에 여러 번 적용되지 않게 한다.

## 28. 스테이지

하나의 스테이지는 고정된 벽돌 패턴으로 구성한다.

### 스테이지 시작

* 막대 초기 위치 설정
* 공 1개를 막대 위에 배치
* 벽돌 패턴 생성
* 아이템과 이펙트 초기화
* `ready` 상태 진입

### 스테이지 클리어

```text
마지막 벽돌 파괴
→ 스테이지 클리어 판정
→ 공과 아이템 이동 중지
→ 점수 정산
```

스테이지 클리어 처리는 한 번만 실행한다.

v0.1에서는 기본 스테이지 하나만 구현한다.

클리어 후에는 같은 스테이지 재시작 또는 게임 재시작이 가능하다.

## 32. 시간 처리

게임 물리는 delta time 기반으로 처리한다.

```javascript
deltaTime = Math.min(deltaTime, MAX_DELTA_TIME);
```

권장 최대 delta time:

```text
0.033초에서 0.05초
```

탭이 비활성화됐다가 다시 활성화될 때 공이 순간 이동하지 않게 이전 시간을 초기화한다.

일시정지 중에는 물리, 아이템 낙하, 효과 타이머가 진행되지 않아야 한다.

## 33. Canvas 크기와 해상도

CSS 표시 크기와 Canvas 내부 해상도를 분리한다.

* CSS에서 Canvas가 컨테이너를 채움
* JavaScript에서 실제 표시 크기 측정
* devicePixelRatio 반영
* 지나치게 높은 DPR은 성능을 위해 제한 가능
* 화면 회전과 크기 변경에 대응
* 막대와 공 위치를 새 크기의 유효 범위로 보정
* 벽돌 배치가 화면 밖으로 나가지 않게 재계산
* 물리 좌표와 렌더링 좌표 기준을 통일

Canvas 크기 변경으로 벽돌 HP나 게임 진행 상태가 초기화되지 않게 한다.

## 34. 입력 처리

Pointer Events를 우선 사용한다.

지원 입력:

* 모바일 터치
* 마우스
* 스타일러스

### 모바일

* Canvas 좌우 드래그로 막대 이동
* 세로 이동은 막대 조작에 사용하지 않음
* 페이지 스크롤 방지
* 터치 확대와 길게 누르기 방지
* 포인터가 Canvas 밖으로 나가도 정상 종료
* 필요하면 pointer capture 사용

### PC

* Canvas 안에서 마우스 이동으로 막대 이동 가능
* 또는 클릭·드래그 방식 지원
* 공 대기 상태에서 클릭으로 발사 가능

### 입력 충돌 방지

* Pointer Events와 별도의 mouse/touch 이벤트를 동시에 등록하지 않음

* 이벤트 리스너 중복 등록 금지

## 44. 오류 방지 체크리스트

### 초기화

* 모든 스크립트가 순서대로 로드되는가
* `AbyssBreaker` 네임스페이스가 정상 생성되는가
* Canvas와 주요 DOM 요소가 존재하는가
* 게임 루프가 한 번만 시작되는가
* 이벤트 리스너가 한 번만 등록되는가

### 막대

* 막대가 화면 밖으로 나가지 않는가
* 작은 화면에서도 막대가 정상 표시되는가
* 막대 확장 후 경계를 벗어나지 않는가
* 효과 종료 후 기본 너비로 돌아오는가

### 공

* 공 좌표와 속도에 NaN이 발생하지 않는가
* 공이 벽이나 막대 내부에 갇히지 않는가
* 공이 지나치게 수평으로 움직이지 않는가
* 공이 화면 아래에서 정상 제거되는가
* 공이 빠를 때 벽돌을 통과하지 않는가
* 멀티볼 최대 개수가 지켜지는가

### 생명

* 공 하나가 떨어졌을 때 다른 공이 있으면 생명이 감소하지 않는가
* 마지막 공이 떨어졌을 때만 생명이 감소하는가
* 생명 감소가 중복 실행되지 않는가
* 생명이 남으면 공이 막대 위에 재배치되는가
* 생명이 0이면 게임오버가 되는가

### 벽돌

* 같은 벽돌에 한 프레임 중복 피해가 발생하지 않는가
* 파괴된 벽돌이 다시 충돌하지 않는가
* 점수가 중복 지급되지 않는가
* 아이템이 한 번만 드롭되는가
* 마지막 벽돌 파괴 후 클리어가 한 번만 처리되는가

### 아이템

* 아이템이 아래로 정상 낙하하는가
* 막대와 충돌하면 한 번만 적용되는가
* 화면 아래로 나가면 제거되는가
* 멀티볼이 안전한 위치에 생성되는가
* 감속 종료 후 속도가 정상 복구되는가
* 재시작 후 이전 효과가 남지 않는가

### UI

* 전체 페이지 스크롤이 생기지 않는가
* 버튼과 Canvas가 겹치지 않는가
* 오버레이가 숨겨졌을 때 클릭을 막지 않는가
* 오버레이 표시 중 게임 입력이 차단되는가
* PC 화면에서 중앙 정렬되는가
* 사용자 표시 문구가 한국어인가

### 저장

* 최고 점수가 저장되는가
* 새로고침 후 최고 점수가 유지되는가
* 저장 데이터가 손상되어도 기본 상태로 실행되는가
