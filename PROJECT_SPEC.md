# Abyss Breaker Project Spec

## 현재 버전

- 통합 버전: v1.0.0
- 장르: 실시간 막대 반사형 벽돌깨기
- 실행 방식: 외부 라이브러리, ES Module, npm, 빌드 도구 없이 정적 HTML/CSS/JavaScript 실행
- 저장 방식: localStorage `abyssBreaker.save.v3`

## 핵심 게임 루프

1. 로비에서 게임 모드, 클래스, 영구 성장, 기록, 업적, 설정을 확인한다.
2. 런 시작 후 패들로 공을 반사해 벽돌과 보스를 파괴한다.
3. 스테이지 클리어 시 런 능력을 선택한다.
4. 보스 보상 구간에서는 유물을 선택한다.
5. 게임오버 또는 런 클리어 시 심연석, 기록, 업적을 정산한다.
6. 심연석으로 클래스와 영구 성장을 해금/구매한다.

## v1.0.0 필수 기능

### 게임 모드

- 표준 모드: 1~10스테이지 클리어형 기본 모드
- 무한 모드: 10스테이지 이후 계속 진행, 보스 주기 반복, 난이도/보상 스케일링
- 단일 생명 도전: 생명 1개, 생명 회복/최대 생명 능력 제외
- 아이템 봉인 도전: 드롭 아이템 없음, 아이템 관련 능력/유물 제외
- 고속 심연 도전: 공 속도 증가, 감속 능력/아이템 제외

### 해금

- 표준 모드 첫 클리어 후 무한 모드와 도전 모드를 해금한다.
- 잠긴 모드는 로비에서 선택할 수 없다.

### 업적

- 업적은 저장 상태로 중복 보상을 차단한다.
- 업적 보상은 심연석으로 지급한다.
- v1.0.0 기준 15개 업적을 제공한다.

### 기록

- 총 런, 총 클리어, 누적 벽돌/아이템/보스 기록을 저장한다.
- 표준 모드 최고 점수, 최고 스테이지, 클리어 수, 최단 클리어 시간을 저장한다.
- 무한 모드 최고 점수, 최고 스테이지, 보스 처치 기록을 저장한다.
- 도전 모드별 최고 점수, 최고 스테이지, 클리어 여부, 클리어 수, 최단 클리어 시간을 저장한다.
- 클래스별 최고 점수를 저장한다.

### 저장 스키마 v3

- `schemaVersion: 3`
- 기존 v1/v2 저장 데이터는 v3 기본 구조로 마이그레이션한다.
- 저장 데이터는 내보내기, 불러오기, 전체 초기화를 지원한다.

### 설정

- 사운드 켜기/끄기
- 진동 켜기/끄기
- 효과 감소 켜기/끄기

## 안정성 원칙

- 게임 루프는 `main.js`의 단일 `requestAnimationFrame` 루프만 사용한다.
- 런 보상은 `runRewardGranted`로 1회만 지급한다.
- 저장 마이그레이션은 누락 필드를 기본값으로 보정한다.
- 모드 전환은 `Data.MODES`에 정의된 상태만 사용한다.
- 모든 런 수치 계산은 NaN이 되지 않도록 기본값과 clamp를 사용한다.
